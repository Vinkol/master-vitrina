## 🏗 Архитектура приложения

Проект спроектирован по классической трехзвенной схеме и полностью контейнеризирован:

┌────────────────────────────────────────────────────────┐│               Telegram Mini App (Frontend)             ││   React 19 / Vite SPA ➔ Zustand State ➔ Axios Client  │└───────────────────────────┬────────────────────────────┘│ API-запросы (JWT Bearer)▼┌────────────────────────────────────────────────────────┐│                Python FastAPI (Backend API)            ││   SQLAlchemy ORM ➔ Alembic Migrations ➔ JWT Auth Guard │└───────────────────────────┬────────────────────────────┘│ Драйвер asyncpg▼┌────────────────────────────────────────────────────────┐│              PostgreSQL 16 (Data Layer)                ││       Таблицы: user_master, client, services...        │└────────────────────────────────────────────────────────┘
### 💻 Frontend Component (Клиентская часть)
- Развёрнут на **React 19** и **Vite** для мгновенной отрисовки интерфейса внутри WebView мессенджера [index: 0.1.2, 0.1.3].
- Локальное состояние и кэширование данных координируются через **Zustand** (разделение на изолированные слайсы логики) [index: 0.1.3, 0.1.32].
- Стилизация построена на базе **Tailwind CSS v4** с использованием аппаратных анимаций выезда шторок (*Bottom Sheets*) [index: 0.1.2, 0.1.13, 0.1.17].
- Интегрирован **Telegram WebApp SDK**: нативная работа с кнопкой `MainButton`, тактильные микровибрации смартфона через `HapticFeedback` и бесшовное сворачивание TWA при переходе в чат техподдержки [index: 0.1.18, 0.1.30, 0.1.32, 0.1.34].

### 🐍 Backend Component (Серверная часть)
- Высокопроизводительный асинхронный API-сервер на **Python 3.11+ / FastAPI** [index: 0.1.1, 0.1.10, 0.1.11].
- Безопасность: авторизация по Telegram ID с выдачей защищенных JWT-токенов доступа [index: 0.1.11, 0.1.28].
- Работа с базой организована через асинхронный движок **SQLAlchemy (asyncpg)** [index: 0.1.9, 0.1.11].
- Контроль структуры таблиц и версионирование схемы данных реализованы через **Alembic** [index: 0.1.1, 0.1.3].

### 🛢 Data Layer (Слой данных)
- Реляционная СУБД **PostgreSQL 16 (Alpine)** [index: 0.1.10].
- Архитектура таблиц включает автоматическую генерацию UUID (`gen_random_uuid()`), хранение сложных конфигураций рабочих дней в массивах `JSONB` и каскадное удаление зависимых записей (`ON DELETE CASCADE`) [index: 0.1.6, 0.1.7, 0.1.8].

## ⚡ Ключевые Инженерные Решения

- **Многозадачная техподдержка:** При вызове `openTelegramLink` приложение нативно сворачивается, позволяя отправить скриншот ошибки разработчику. При возвращении стейт и открытые экраны полностью сохраняются [index: 0.1.32].
- **Унифицированный UI-код:** Компонент `MenuRowButton` инкапсулирует стили подложек иконок и автоматически подстраивается под тип контента (Base64 аватарка профиля или векторный SVG-силуэт) [index: 0.1.16].
- **Гибридный UX (Валидация платформ):** Приложение определяет среду запуска [index: 0.1.37]. Внутри Telegram управление бронированием перехватывает нативная нижняя кнопка мессенджера; в обычных браузерах на ПК рендерится стандартная кнопка интерфейса [index: 0.1.34, 0.1.37].
- **Синхронная CRM-аналитика:** Расчет лояльности перенесен на уровень бизнес-логики. Статус **«Постоянный клиент»** присваивается в списках пользователей на экранах админки строго при фиксации бэкендом **3 и более успешных посещений** [index: 0.1.9].

## 🛠 Контур автоматизации и контроля качества (CI/CD)

1. **Тестовая среда (Vitest):** Быстрый запуск юнит- и интеграционных тестов в среде `jsdom`. Написаны сценарии для проверки моков Telegram SDK и фолбэков [index: 0.2.14].
2. **Git Hooks (Husky + lint-staged):** Робот перехватывает команду `git commit` [index: 0.1.0]. Он изолирует измененные файлы, форматирует их через `Prettier` по единому стандарту (`.prettierrc`) и запускает тесты [index: 0.1.2, 0.1.11]. Если тест упал — коммит блокируется.
3. **Строгий TypeScript:** Конфигурация `tsconfig.json` переведена на современный стандарт разрешения модулей `bundler`, включены строгие правила типизации (запрет `any`, контроль switch-case) [index: 0.2.14].
